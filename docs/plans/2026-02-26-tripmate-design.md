# 智能旅游助手 App - 设计文档

**日期**: 2026-02-26  
**版本**: v1.0

---

## 1. 项目概述

### 项目名称
**TripMate** - 智能旅游助手

### 项目目标
为个人用户提供一款集合 AI 智能行程规划、实时导航导览、美食推荐和旅行安全保障于一体的移动应用，让旅游规划更轻松，旅途更便捷。

### 目标用户
- 热爱旅行的个人用户
- 喜欢自由行但不想花大量时间做攻略的游客
- 对旅行体验有品质要求的用户

---

## 2. 功能规划

### 2.1 核心功能 (MVP)

| 功能模块 | 描述 | 优先级 |
|----------|------|--------|
| **AI 智能行程规划** | 用户输入目的地、旅行天数、预算、偏好风格，AI 自动生成完整行程安排 | P0 |
| **实时导航与导览** | 基于地图的位置服务，提供景点导航路线规划 | P0 |
| **美食推荐** | 根据用户位置和口味偏好推荐当地特色美食餐厅 | P1 |
| **旅行安全保障** | 紧急联系人、天气提醒、旅行安全提示 | P1 |
| **风格化行程** | 提供不同风格的旅行计划：穷游/舒适/豪华/亲子/情侣等 | P0 |

### 2.2 功能详细说明

#### AI 智能行程规划
- 输入表单：目的地、出行日期、人数、预算、旅行风格、特殊需求
- 输出：每日行程安排，包含景点、餐厅、住宿建议、时间规划
- 支持行程修改和手动调整

#### 实时导航与导览
- 集成地图 SDK（高德/百度地图）
- 景点位置标注和路线规划
- 支持离线地图下载

#### 美食推荐
- 基于位置推荐附近美食
- 支持菜系筛选、价位筛选、评分排序
- 展示餐厅地址、营业时间、用户评价

#### 旅行安全保障
- 紧急联系人设置
- 目的地天气预警推送
- 旅行保险提醒
- 当地紧急电话展示

## 3. 技术架构

### 3.1 技术选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **框架** | Flutter 3.x | 跨平台移动开发框架 |
| **语言** | Dart 3.x | Flutter 官方语言 |
| **状态管理** | Riverpod / GetX | 轻量级状态管理 |
| **本地存储** | Hive / SQLite | 离线数据存储 |
| **网络** | Dio | HTTP 客户端 |
| **地图** | amap_flutter_map | 高德地图 Flutter 插件 |
| **AI 对接** | OpenAI API / Claude API | AI 行程生成 |

### 3.2 项目结构

```
lib/
├── main.dart                 # 应用入口
├── app/                      # App 级别配置
│   ├── routes/               # 路由配置
│   ├── theme/                # 主题配置
│   └── constants/            # 常量定义
├── core/                     # 核心功能
│   ├── network/              # 网络请求
│   ├── storage/              # 本地存储
│   └── utils/                 # 工具类
├── features/                 # 功能模块
│   ├── home/                 # 首页
│   ├── planning/             # 行程规划
│   ├── navigation/           # 导航导览
│   ├── food/                 # 美食推荐
│   └── safety/               # 安全保障
└── shared/                   # 共享组件
    ├── widgets/              # 通用组件
    └── models/               # 数据模型
```

## 4. UI/UX 设计

### 4.1 页面结构

| 页面 | 功能 |
|------|------|
| **首页** | 功能入口、常用目的地快捷访问 |
| **行程规划页** | 输入旅行偏好，生成行程 |
| **行程详情页** | 查看和调整生成的行程 |
| **地图导航页** | 实时地图和导航 |
| **美食推荐页** | 附近美食列表和详情 |
| **我的页面** | 收藏、历史行程、设置 |

### 4.2 设计风格

- **风格**: 现代简约 + 旅行元素
- **主色调**: 蓝色 (#2196F3) 代表天空和海洋，绿色 (#4CAF50) 代表自然
- **字体**: 思源黑体 / Roboto
- **图标**: Material Icons + 自定义旅行图标

## 5. 数据流设计

### 5.1 AI 行程生成流程

```
用户输入 → 表单验证 → 调用 AI API → 解析响应 → 生成行程数据 → 本地存储 → 展示
```

### 5.2 关键数据模型

- **Trip**: 旅行计划基本信息
- **DayPlan**: 每日行程安排
- **Spot**: 景点信息
- **Restaurant**: 餐厅信息
- **UserPreferences**: 用户偏好设置

## 7. 风险和挑战

| 风险 | 应对措施 |
|------|----------|
| AI API 响应不稳定 | 添加加载状态和重试机制 |
| 地图服务需要 API Key | 使用免费配额或模拟数据 |
| 离线功能复杂 | MVP 阶段先做在线版 |
